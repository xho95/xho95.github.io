> Apple 에서 공개한 [The Swift Programming Language (Swift 5.4)](https://docs.swift.org/swift-book/) 책의 [Types](https://docs.swift.org/swift-book/ReferenceManual/Types.html) 부분[^Types]을 번역하고, 설명이 필요한 부분은 주석을 달아서 정리한 글입니다. 전체 번역은 [Swift 5.4: Swift Programming Language (스위프트 프로그래밍 언어)]({% post_url 2017-02-28-The-Swift-Programming-Language %}) 에서 확인할 수 있습니다.

## Types (타입)

스위프트에는, 두 가지 종류의 타입: '이름 붙은 (named) 타입' 과 '복합 (compound) 타입' 이 있습니다. _이름 붙은 타입 (named type)_ 은 정의할 때 특정 이름을 부여할 수 있는 타입입니다. '이름 붙은 타입' 은 클래스, 구조체, 열거체, 그리고 프로토콜을 포함합니다. 예를 들어, 사용자가 `MyClass` 라는 이름을 붙여서 정의한 클래스의 인스턴스는 `MyClass` 라는 타입을 가집니다. 사용자가 이름 붙여 정의한 타입에 더하여, 스위프트 표준 라이브러리는 공통적으로 많이 사용하는, 배열, 딕셔너리, 그리고 옵셔널 값의 표현을 포함한, '이름 붙은 타입' 을 정의합니다.

다른 언어에서 '기초' 또는 '근원' 이라고 고려하는-수, 문자, 그리고 문자열 같은-'자료 (data) 타입' 은 실제로는, 스위프트 표준 라이브러리에서 구조체로 정의하고 구현한, '이름 붙은 타입' 입니다. '이름 붙은 타입' 이기 때문에, [Extensions (익스텐션; 확장)]({% post_url 2016-01-19-Extensions %}) 과 [Extension Declaration (익스텐션 선언)]({% post_url 2020-08-15-Declarations %}#extension-declaration-익스텐션-선언) 에서 논한 것처럼, '익스텐션 선언' 을 사용하여, 프로그램 필요에 적합하도록 동작을 확장할 수 있습니다.

_복합 타입 (compound types)_ 은, 스위프트 언어 자체에서 정의한, '이름 없는 타입' 입니다. 복합 타입은: '함수 타입' 과 '튜플 타입' 두 가지가 있습니다. '복합 타입' 은 '이름 붙은 타입' 과 다른 '복합 타입' 을 담을 수도 있습니다. 예를 들어, `(Int, (Int, Int))` 튜플 타입은 두 원소를 담고 있습니다: 첫 번째는 '이름 붙은 타입' 인 `Int` 이고, 두 번째는 또 다른 '복합 타입' 인 `(Int, Int)` 입니다.

'이름 붙은 타입' 이나 '복합 타입' 주위에 괄호를 칠 수 있습니다. 하지만, 타입 주위에 괄호를 추가하는 건 어떤 효과도 없습니다. 예를 들어, `(Int)` 는 `Int` 와 '동치 (equivalent)' 입니다.

이 장은 스위프트 언어에서 정의한 타입을 논하고 스위프트의 '타입 추론 동작' 에 대해서 설명합니다.

### Type Annotation (타입 보조 설명)

_타입 보조 설명 (type annotation)_ 은 변수나 표현식의 타입을 명시적으로 지정합니다. 타입 보조 설명은, 다음 예제에서 보는 것처럼, '콜론 (`:`)' 으로 시작하고 타입으로 끝납니다:

```swift
let someTuple: (Double, Double) = (3.14159, 2.71828)
func someFunction(a : Int) { /* ... */ }
```

첫 번째 예제의, `someTuple` 표현식은 `(Double, Double)` 이라는 튜플 타입을 갖도록 지정합니다. 두 번째 예제에서, `someFunction` 함수의 `a` 라는 매개 변수는 `Int` 타입을 갖도록 지정합니다.

'타입 보조 설명' 은 타입 앞에 '타입 특성 (type attributes) 목록' 을 선택 사항으로 담고 있을 수 있습니다.

> GRAMMAR OF A TYPE ANNOTATION 부분 생략 - [링크](https://docs.swift.org/swift-book/ReferenceManual/Types.html#ID446)

### Type Identifier (타입 식별자)

_타입 식별자 (type identifier)_ 는 '이름 붙은 타입' 아니면 '이름 붙은 또는 복합 타입의 타입 별명 (type alias)' 을 참조합니다.

대부분의 시간 동안, 타입 식별자는 식별자와 똑같은 이름을 가진 '이름 붙은 타입' 을 직접 참조합니다. 예를 들어, `Int` 는 `Int` 라는 '이름이 붙은 타입' 을 직접 참조하는 '타입 식별자' 이며, `Dictionary <String, Int>` 라는 타입 식별자는 `Dictionary <String, Int>` 라는 '이름이 붙은 타입' 을 직접 참조합니다.

'타입 식별자' 가 똑같은 이름의 타입을 참조하지 않는 경우는 두 가지가 있습니다. 첫 번째 경우는, 타입 식별자가 '이름 붙은 또는 복합 타입의 타입 별명' 을 참조하는 것입니다. 구체적인 예를 들어 보면, 아래 예제의, '타입 보조 설명' 에 사용하는 `Point` 는 `(Int, Int)` 라는 '튜플 타입' 을 참조합니다.

```swift
typealias Point = (Int, Int)
let origin: Point = (0, 0)
```

두 번째 경우는, 타입 식별자가 다른 모듈에서 선언한 또는 다른 타입 안에서 중첩한 '이름 붙은 타입' 을 참조하면서 '점 구문 (`.`)' 을 사용하는 것입니다. 예를 들어, 다음 코드에 있는 타입 식별자는 `ExampleModule` 모듈에서 선언한 `MyType` 이라는 '이름 붙은 타입' 을 참조합니다.

```swift
var someValue: ExampleModule.MyType
```

### Tuple Type (튜플 타입)

_튜플 타입 (tuple type)_ 은 쉼표로 구분된 타입 목록을, 괄호로 감싼 것입니다.

튜플 타입을 함수의 반환 타입으로 사용하면 함수가 다중 값을 가지는 단일 튜플을 반환하도록 해줍니다. 튜플 타입의 원소에도 이름을 붙일 수 있으며 이 이름을 사용하여 개별 원소의 값을 참조할 수도 있습니다. 원소 이름은 식별자와 그 바로 뒤의 '콜론 (`:`)' 으로 구성됩니다. 이러한 두 개의 특성을 모두 증명해 보이는 예제는, [Functions with Multiple Return Values (반환 값이 여러 개인 함수)]({% post_url 2020-06-02-Functions %}#functions-with-multiple-return-values-반환-값이-여러-개인-함수) 를 참고하기 바랍니다.

튜플 타입의 원소가 이름을 가질 때, 해당 이름은 타입의 일부입니다.

```swift
var someTuple = (top: 10, bottom: 12)  // someTuple 의 타입은 (top: Int, bottom: Int) 입니다.
someTuple = (top: 4, bottom: 42) // OK: 이름이 일치합니다.
someTuple = (9, 99)              // OK: 이름이 추론됩니다.
someTuple = (left: 5, right: 5)  // Error: 이름이 일치하지 않습니다.
```

모든 튜플 타입은 둘 이상의 타입을 가지는데, `()` 라는, 빈 튜플 타입에 대한 '타입 별명 (type alias)' 인 `Void` 는 가질 수 없습니다.

### Function Type (함수 타입)

_함수 타입 (funtion type)_ 은 함수, 메소드, 또는 클로저의 타입을 나타내는 것으로 '화살표 (`->`)' 로 구분된 매개 변수 타입과 반환 타입으로 구성됩니다:

(`parameter type`) -> `return type`

_매개 변수 타입 (parameter type)_ 은 쉼표로-구분된 타입의 목록입니다. _반환 타입 (return type)_ 은 튜플 타입일 수 있기 때문에, 함수 타입은 다중 값을 반환하는 함수와 메소드를 지원합니다.

(`T` 가 어떤 타입도 되는) `() -> T` 라는 함수 타입의 매개 변수에 `autoclosure` 특성을 적용하면 호출 측에서 암시적으로 클로저를 생성할 수 있습니다. 이는 함수를 호출할 때 명시적인 클로저를 작성하지 않고도 표현식의 값-평가를 지연하는 구문상의 편의 방법을 제공하는 것입니다. 자동 클로저 함수 타입 매개 변수에 대한 예제는, [Autoclosures (자동 클로저)]({% post_url 2020-03-03-Closures %}#autoclosures-자동-클로저) 를 참고하기 바랍니다.

함수 타입은 _매개 변수 타입 (parameter type)_ 에 '가변 매개 변수 (variadic parameter)' 를 가질 수 있습니다. 구문 표현으로 보면, 가변 매개 변수는, `Int...` 에서와 같이, 기본 타입 이름과 그 바로 뒤의 세 점 (`...`) 으로 구성됩니다. 가변 매개 변수는 기본 타입 이름의 원소를 가지고 있는 배열인 것처럼 취급합니다. 예를 들어 보면, 가변 매개 변수 `Int...` 는 `[Int]` 인 것으로 취급합니다. 가변 매개 변수를 사용하는 예제에 대해서는, [Variadic Parameters (가변 매개 변수)]({% post_url 2020-06-02-Functions %}#variadic-parameters-가변-매개-변수) 를 참고하기 바랍니다.

'입-출력 매개 변수 (in-out parameter)' 를 지정하려면, 매개 변수 타입에 `inout` 키워드로 접두사를 붙입니다. 가변 매개 변수나 반환 타입을  `inout` 키워드로 표시할 수는 없습니다. 입-출력 매개 변수는 [In-Out Parameters (입-출력 매개 변수)]({% post_url 2020-06-02-Functions %}#in-out-parameters-입-출력-매개-변수) 에서 설명합니다.

함수 타입이 단 하나의 매개 변수를 가지는데 그 매개 변수의 타입이 튜플 타입인 경우라면, 함수 타입을 작성할 때 그 튜플 타입은 반드시 괄호로 묶어야 합니다. 예를 들어, `((Int, Int)) -> Void` 는 튜플 타입 `(Int, Int)` 라는 단일 매개 변수를 받아서 어떤 값도 반환하지 않는 함수의 타입입니다. 이와 대조해서, 괄호가 없는, `(Int, Int) -> Void` 는 두 개의 `Int` 매개 변수를 받아서 어떤 값도 반환하지 않는 함수의 타입입니다. 마찬가지로, `Void` 는 `()` 에 대한 '타입 별명 (type alias)' 이기 때문에, 함수 타입 `(Void) -> Void` 는-빈 튜플인 단일 인자를 받아 들이는 함수인-`(()) -> ()` 와 같습니다. 이러한 타입은-아무 인자를 받지 않는 함수인-`() -> ()` 와 같은 것이 아닙니다.

함수와 메소드에 있는 인자 이름은 연관된 함수 타입의 일부가 아닙니다.[^argument-name-in-function] 예를 들면 다음과 같습니다:

```swift
func someFunction(left: Int, right: Int) {}
func anotherFunction(left: Int, right: Int) {}
func functionWithDifferentLabels(top: Int, bottom: Int) {}

var f = someFunction // f 의 타입은 (Int, Int) -> Void 인 것으로, (left: Int, right: Int) -> Void 인 것이 아닙니다.
f = anotherFunction              // OK
f = functionWithDifferentLabels  // OK

func functionWithDifferentArgumentTypes(left: Int, right: String) {}
f = functionWithDifferentArgumentTypes     // Error

func functionWithDifferentNumberOfArguments(left: Int, right: Int, top: Int) {}
f = functionWithDifferentNumberOfArguments // Error
```

'인자 이름표 (argument labels)' 는 함수 타입의 일부가 아니기 때문에, 함수 타입을 작성할 때는 생략합니다.

```swift
var operation: (lhs: Int, rhs: Int) -> Int     // Error
var operation: (_ lhs: Int, _ rhs: Int) -> Int // OK
var operation: (Int, Int) -> Int               // OK
```

함수 타입이 하나 이상의 '화살표 (`->`)' 를 포함하는 경우, 함수 타입은 오른쪽에서 왼쪽으로 그룹을 짓습니다. 예를 들어, 함수 타입 `(Int) -> (Int) -> Int` 는 `(Int) -> ((Int) -> Int)` 라고 이해합니다-즉, `Int` 를 받아서 `Int` 를 받고 반환하는 다른 함수를 반환하는 함수라는 것입니다.

에러를 던지거나 다시 던질 수 있는 함수 타입은 반드시 `throws` 키워드로 표시해야 합니다. `throws` 키워드는 함수 타입의 일부이며, '던지지 않는 함수 (nonthrowing functions)' 은 '던지는 함수 (throwing functions)' 의 하위 타입입니다. 그 결과, '던지지 않는 함수' 를 던지는 함수와 같은 위치에 사용할 수 있습니다. '던지는 함수' 와 '다시 던지는 함수 (rethrowing functions)' 는 [Throwing Functions and Methods (던지는 함수 및 메소드)]({% post_url 2020-08-15-Declarations %}#throwing-functions-and-methods-던지는-함수-및-메소드) 및 [Rethrowing Functions and Methods (다시 던지는 함수 및 메소드)]({% post_url 2020-08-15-Declarations %}#rethrowing-functions-and-methods-다시-던지는-함수-및-메소드) 에서 설명합니다.

#### Restrictions for Nonescaping Closures (벗어나지 않는 클로저에 대한 제약 조건)

매개 변수가 '벗어나지 않는 함수' 이면 속성이나, 변수, 또는 `Any` 타입의 상수에 저장할 수 없는데, 이는 값이 벗어날 수 있기 때문입니다.

매개 변수가 '벗어나지 않는 함수' 이면 다른 '벗어나지 않는 함수' 매개 변수의 인자로 전달할 수 없습니다. 이러한 제약 조건은 스위프트가 실행 시간 대신에 컴파일 시간에 더 많은 메모리 접근 충돌 검사를 수행하도록 돕습니다. 예를 들면 다음과 같습니다:

```swift
let external: (() -> Void) -> Void = { _ in () }
func takesTwoFunctions(first: (() -> Void) -> Void, second: (() -> Void) -> Void) {
  first { first {} }       // 에러 (Error)
  second { second {} }     // 에러 (Error)

  first { second {} }      // 에러 (Error)
  second { first {} }      // 에러 (Error)

  first { external {} }    // OK
  external { first {} }    // OK
}
```

위의 코드에서, `takesTwoFunctions(first:second:)` 의 두 매개 변수는 모두 함수입니다. 어느 매개 변수도 `@escaping` 으로 표시하지 않았으며, 그 결과 둘 다 '벗어나지 않는' 것들입니다.

위 예제에서 "에러 (Error)" 로 표시한 네 개의 함수 호출은 컴파일러 에러를 야기합니다. `first` 와 `second` 매개 변수는 '벗어나지 않는 함수' 이기 때문에, 다른 벗어나지 않는 함수 매개 변수의 인자로 전달할 수 없습니다. 이와는 대조적으로, "OK" 로 표시한 두 개의 함수 호출은 컴파일러 에러를 야기하지 않습니다. 이 함수 호출들은 `external` 이 `takesTwoFunctions(first:second:)` 의 매개 변수 중 하나가 아니기 때문에 제약 조건을 위반하지 않습니다.

이런 제약 조건을 피하고자 할 경우, 매개 변수 중 하나를 '벗어나는' 것으로 표시하거나, 아니면 `withoutActuallyEscaping(_:do:)` 함수를 사용하여 '벗어나지 않는 함수 매개 변수' 를 '벗어나는 함수' 로 임시적으로 변환하도록 합니다. 메모리 접근의 충돌을 피하는 것에 대한 정보는, [Memory Safety (메모리 안전성)]({% post_url 2020-04-07-Memory-Safety %}) 을 참고하기 바랍니다.

### Array Type (배열 타입)

스위프트 언어는 스위프트 표준 라이브러리의 `Array<Element>` 타입에 대해서 다음과 같은 '수월한 구문 표현 (systatic sugar)' 을 제공합니다:

[`type`]

다르게 말해서, 다음의 두 선언은 '동치 (equivalent)' 입니다:

```swift
let someArray: Array<String> = ["Alex", "Brian", "Dave"]
let someArray: [String] = ["Alex", "Brian", "Dave"]
```

두 경우 모두, `someArray` 라는 상수를 문자열 배열로 선언합니다. 배열의 원소는 대괄호 안에 유효한 색인 값을 지정하는 첨자 연산을 통하여 접근할 수 있습니다: `someArray[0]` 은 색인이 '0' 인 원소인, `"Alex"` 를 참조합니다.

대괄호 쌍을 중첩하면 '다-차원 (multidimensional)' 배열을 생성할 수 있으며, 여기서 원소의 기본 타입 이름은 가장 안쪽의 대괄호 쌍에서 가지게 됩니다. 예를 들어, 세 쌍의 대괄호를 사용하면 삼-차원 정수 배열을 생성할 수 있습니다:

```swift
var array3D: [[[Int]]] = [[[1, 2], [3, 4]], [[5, 6], [7, 8]]]
```

다-차원 배열에 있는 원소에 접근할 때, 가장-왼쪽의 '첨자 연산 색인' 은 가장 바깥쪽 배열에 있는 해당 색인의 원소를 참조합니다. 오른쪽으로 그 다음의 첨자 연산 색인은 한 단계 중첩된 배열에 있는 해당 색인의 원소를 참조합니다. 이런 식으로 계속합니다. 이것이 의미하는 것은 위 예제에서, `array3D[0]` 는 `[[1, 2], [3, 4]]` 를 참조하고, `array3D[0][1]` 은 `[3, 4]` 를 참조하며, `array3D[0][1][1]` 은 값 '4' 를 참조한다는 것입니다.

스위프트 표준 라이브러리의 `Array` 타입에 대한 자세한 논의는, [Arrays (배열)]({% post_url 2016-06-06-Collection-Types %}#arrays-배열) 를 참고하기 바랍니다.

### Dictionary Type (딕셔너리 타입)

스위프트 언어는 스위프트 표준 라이브러리의 `Dictionary<Key, Value>` 타입에 대해서 다음과 같은 '수월한 구문 표현 (systatic sugar)' 을 제공합니다:

[`key type`: `value type`]

다르게 말해서, 다음의 두 선언은 '동치 (equivalent)' 입니다:

```swift
let someDictionary: [String: Int] = ["Alex": 31, "Paul": 39]
let someDictionary: Dictionary<String, Int> = ["Alex": 31, "Paul": 39]
```

두 경우 모두, `someDictionary` 라는 상수를 키는 문자열이고 값은 정수인 '딕셔너리' 라고 선언합니다.

딕셔너리의 값은 대괄호 안에 관련된 키를 지정하는 첨자 연산을 통하여 접근할 수 있습니다: `someDictionary["Alex"]` 는 `"Alex"` 라는 키와 결합되어 있는 값을 참조합니다. 첨자 연산은 딕셔너리의 값 타입에 대한 옵셔널 값을 반환합니다. 딕셔너리가 지정한 키를 가지고 있지 않으면, 첨자 연산이 `nil` 을 반환합니다.

딕셔너리의 키 타입은 스위프트 표준 라이브러리의 `Hashable` 프로토콜을 반드시 준수해야 합니다.

스위프트 표준 라이브러리의 `Dictionary` 타입에 대한 자세한 논의는, [Dictionaries (딕셔너리)]({% post_url 2016-06-06-Collection-Types %}#dictionaries-딕셔너리) 을 참고하기 바랍니다.

> GRAMMAR OF A DICTIONARY TYPE 부분 생략 - [링크](https://docs.swift.org/swift-book/ReferenceManual/Types.html#ID451)

### Optional Type (옵셔널 타입)

스위프트 언어는 접미사 `?` 를, 스위프트 표준 라이브러리에서 정의한, '이름 있는 타입' 인 `Optional<Wrapped>` 에 대한 '수월한 구문 표현 (syntactic sugar)' 으로 정의합니다. 다르게 말해서, 다음의 두 선언은 '동치 (equivalent)' 입니다:

```swift
var optionalInteger: Int?
var optionalInteger: Optional<Int>
```

두 경우 모두, `optionalInteger` 라는 변수가 '옵셔널 정수' 라는 타입을 가진다고 선언합니다. 타입과 `?` 사이에 공백은 전혀 없을 거라는 점에 주목하기 바랍니다.

`Optional<Wrapped>` 타입은, 있을 수도 없을 수도 있는 값을 나타내기 위해 사용하는, `none` 과 `some(Wrapped)` 라는, 두 개의 'case 값' 이 있는 열거체입니다. 어떤 타입이든지 옵셔널 타입이라고 명시적으로 선언 (또는 암시적으로 변환) 할 수 있습니다. 옵셔널 변수나 속성을 선언할 때 초기 값을 제공하지 않으면, 이 값은 자동적으로 `nil` 로 기본 설정됩니다.

옵셔널 타입의 인스턴스가 값을 가지고 있는 경우, 아래에 보인 것처럼, 접미사 연산자 `!` 을 사용하여 해당 값에 접근할 수 있습니다:

```swift
optionalInteger = 42
optionalInteger! // 42
```

`nil` 값을 가지고 있는 옵셔널의 포장을 풀려고 `!` 연산자를 사용하면 그 결과는 실행 시간 에러가 됩니다.

'옵셔널 연쇄 (optional chaining)' 및 '옵셔널 연결 (optional binding)' 을 사용하면 옵셔널 표현식에서 조건부로 연산을 수행할 수도 있습니다. 값이 `nil` 이면, 아무런 연산을 수행하지 않으므로 실행 시간 에러를 일으키지 않습니다.

더 많은 정보 및 옵셔널 타입을 사용하는 방법을 보여주는 예제를 보려면, [Optionals (옵셔널)]({% post_url 2016-04-24-The-Basics %}#optionals-옵셔널) 을 참고하기 바랍니다.

### Implicitly Unwrapped Optional Type (암시적으로 포장이 풀리는 옵셔널 타입)

스위프트 언어는 접미사 `!` 를, 스위프트 표준 라이브러리에서 정의한, '이름 있는 타입' 인 `Optional<Wrapped>` 에 대한 '수월한 구문 표현 (syntactic sugar)' 이면서, 접근할 때 자동으로 포장이 풀린다는 추가적인 동작을 하도록 정의합니다. `nil` 값을 가지고 있는 '암시적으로 포장이 풀리는 옵셔널' 을 사용하려고 하면, 실행 시간 에러를 보게 될 것입니다. 암시적으로 포장을 푸는 동작을 제외한다면, 다음의 두 선언은 '동치 (equivalent)' 입니다:

```swift
var implicitlyUnwrappedString: String!
var explicitlyUnwrappedString: Optional<String>
```

타입과 `!` 사이에 공백은 전혀 없을 거라는 점에 주목하기 바랍니다.

'암시적인 포장 풀기 (implicit unwrapping)' 는 해당 유형을 가지고 있는 선언의 의미를 바꾸기 때문에, 튜플 타입 또는 제네릭 타입 안에 중첩된 옵셔널 타입-딕셔너리나 배열의 원소 타입-같은 것은 '암시적으로 포장이 풀리는' 것이라고 표시할 수 없습니다. 예를 들면 다음과 같습니다:

```swift
let tupleOfImplicitlyUnwrappedElements: (Int!, Int!)  // Error
let implicitlyUnwrappedTuple: (Int, Int)!             // OK

let arrayOfImplicitlyUnwrappedElements: [Int!]        // Error
let implicitlyUnwrappedArray: [Int]!                  // OK
```

'암시적으로 포장이 풀리는 옵셔널' 은 옵셔널 값과 같은 `Optional<Wrapped>` 타입을 가지기 때문에, 코드에서 옵셔널을 사용할 수 있는 것과 같은 모든 위치에서 '암시적으로 포장이 풀리는 옵셔널' 을 사용할 수 있습니다. 예를 들어, '암시적으로 포장이 풀리는 옵셔널' 의 값을 변수, 상수, 및 옵셔널인 속성에 할당할 수 있으며, 그 반대로도 됩니다.

옵셔널에서 처럼, 암시적으로 포장이 풀리는 옵셔널 변수나 속성을 선언할 때 초기 값을 제공하지 않으면, 이 값은 자동적으로 `nil` 로 기본 설정됩니다.

'암시적으로 포장이 풀리는 옵셔널 표현식' 에 대한 조건부 연산을 수행하려면 '옵셔널 연쇄 (optional chaining)' 를 사용합니다. 값이 `nil` 이면, 아무런 연산을 수행하지 않으므로 실행 시간 에러를 일으키지 않습니다.

암시적으로 포장이 풀리는 옵셔널 타입에 대한 더 많은 정보는, [implicitly Unwrapped Optionals (암시적으로 포장이 풀리는 옵셔널)]({% post_url 2016-04-24-The-Basics %}#implicitly-unwrapped-optionals-암시적으로-포장이-풀리는-옵셔널) 를 참고하기 바랍니다.

### Protocol Composition Type (프로토콜 합성 타입)

_프로토콜 합성 타입 (protocol composition type)_ 은 지정한 프로토콜 목록에 있는 각각의 프로토콜을 준수하는 타입, 또는 주어진 클래스의 하위 클래스이면서 지정한 프로토콜 목록에 있는 각각의 프로토콜을 준수하는 타입을 정의합니다. '프로토콜 합성 타입' 은 '타입 보조 설명 (type annotations)' 과, '제네릭 매개 변수 구절', 및 '제네릭 `where` 절' 에서 타입을 지정할 때만 사용하게 됩니다.

프로토콜 합성 타입의 형식은 다음과 같습니다:

`Protocol 1` & `Protocol 2`

프로토콜 합성 타입은, 타입이 준수하려고 하는 각각의 프로토콜을 상속해서 새로운, 이름 있는 프로토콜을 명시적으로 정의하지 않고도, 다중 프로토콜의 필수 조건을 준수하는 타입의 값을 지정하도록 해줍니다. 예를 들어, `ProtocolA`, `ProtocolB`, 및 `ProtocolC` 를 상속하는 새로운 프로토콜을 선언하는 대신 `ProtocolA & ProtocolB & ProtocolC` 라는 프로토콜 합성 타입을 사용할 수 있습니다. 마찬가지로, `SuperClass` 의 하위 클래스이면서 `ProtocolA` 를 준수하는 새로운 프로토콜을 선언하는 대신 `SuperClass & ProtocolA` 를 사용할 수 있습니다.

프로토콜 합성 목록에 있는 각각의 항목은 다음 중 하나에 해당합니다; 이 목록은 최대 하나의 클래스를 가질 수 있습니다:

* 클래스의 이름
* 프로토콜의 이름
* 실제 타입이 '프로토콜 합성 타입' 이거나, 프로토콜, 또는 클래스인 '타입 별명 (type alias)'

프로토콜 합성 타입이 타입 별명을 가지고 있을 때는, 똑같은 프로토콜이 정의에 한 번 이상 있어도 됩니다-중복이면 무시합니다. 예를 들어, 아래 코드의 `PQR` 이라는 정의는 `P & Q & R` 과 '동치 (equivalent)' 입니다.

```swift
typealias PQ = P & Q
typealias PQR = PQ & Q & R
```

### Opaque Type (불투명한 타입)

_불투명한 타입 (opaque type)_ 은, 실제 '구체적으로 고정된 타입 (concrete type)' 을 지정하지 않고도, 프로토콜 또는 '프로토콜 합성 (protocol composition)' 을 준수하는 타입을 정의합니다.

불투명한 타입은 함수나 첨자 연산의 반환 타입으로, 또는 속성의 타입으로 나타납니다. 불투명한 타입은, 배열의 원소 타입이나 옵셔널로 포장한 타입 같은, 튜플 타입 또는 제네릭 타입에서는 나타날 수 없습니다.

불투명한 타입의 형식은 다음과 같습니다:

some `constraint`

_구속 조건 (constraint)_ 는 클래스 타입, 프로토콜 타입, 프로토콜 합성 타입, 또는 `Any` 입니다. 값을 불투명한 타입의 인스턴스로 사용하려면 이것이 목록에 나열된 프로토콜 또는 프로토콜 합성을 준수하는 타입의 인스턴스인 경우, 아니면 목록에 나열된 클래스를 상속받은 경우일 때 뿐입니다. 불투명한 값과 상호 작용하는 코드는 _구속 조건 (constraint)_ 이 정의한 인터페이스에 해당하는 방식으로만 값을 사용할 수 있습니다.

프로토콜 선언은 불투명한 타입을 포함할 수 없습니다. 클래스는 최종이 아닌 메소드의 반환 타입으로 불투명한 타입을 사용할 수 없습니다.

반환 타입으로 불투명한 타입을 사용하는 함수가 반환하는 값은 반드시 단일한 실제 타입을 공유해야 합니다. 반환 타입은 함수의 '일반화한 (generic) 타입 매개 변수' 에 있는 타입을 포함할 수 있습니다. 예를 들어, `someFunction<T>()` 함수는 `T` 타입 또는 `Dictionary<String, T>` 타입의 값을 반환할 수 있습니다.

### Metatype Type (메타타입 타입)

_메타타입 타입 (metatype type)_ 은, 클래스 타입, 구조체 타입, 열거체 타입, 그리고 프로토콜 타입을 포함하는, 어떤 타입에 대한 타입을 참조합니다.

클래스, 구조체, 또는 열거체 타입의 '메타타입 (metatype)' 은 해당 타입의 이름 뒤에 `.Type` 을 붙인 것입니다. 프로토콜 타입의 메타타입-즉 실행 시간에 이 프로토콜을 준수하고 있는 구체적으로 고정된 타입이 아닌 타입-은 해당 프로토콜의 이름 뒤에 `.Protocol` 을 붙인 것입니다. 예를 들어, `SomeClass` 라는 클래스 타입의 메타타입은 `SomeClass.Type` 이고 `SomeProtocol` 이라는 프로토콜의 메타타입은 `SomeProtocol.Protocol` 입니다.

접미사 표현식인 `self` 를 사용하여 타입을 값으로 접근할 수 있습니다. 예를 들어, `SomeClass.self` 는, `SomeClass` 의 인스턴스가 아닌, `SomeClass` 그 자체를 반환합니다. 그리고 `SomeProtocol.self` 는 실행 시간에 `SomeProtocol` 을 준수하고 있는 타입의 인스턴스가 아닌, `SomeProtocol` 그 자체를 반환합니다. 다음 예제에서 보인 것처럼, 타입의 인스턴스에 `type(of:)` 함수를 사용하여 해당 인스턴스의 동적인, 실행 시간 타입을 값으로 접근할 수 있습니다.

```swift
class SomeBaseClass {
  class func printClassName() {
    print("SomeBaseClass")
  }
}
class SomeSubClass: SomeBaseClass {
  override class func printClassName() {
    print("SomeSubClass")
  }
}
let someInstance: SomeBaseClass = SomeSubClass()
// someInstance 의 컴파일-시간 타입은 SomeBaseClass 이고,
// someInstance 의 실행-시간 타입은 SomeSubClass 입니다.
type(of: someInstance).printClassName()
// "SomeSubClass" 를 출력합니다.
```

더 많은 정보는, 스위프트 표준 라이브러리에 있는 [type(of:)](https://developer.apple.com/documentation/swift/2885064-type) 를 참고하기 바랍니다.

해당 타입의 메타타입 값으로 타입의 인스턴스를 생성하려면 '초기자 표현식 (initializer expression)'[^initializer-expression] 을 사용합니다. 클래스 인스턴스인 경우라면, 반드시 호출되는 초기자를 `required` 키워드로 표시하거나 아니면 전체 클래스를 `final` 키워드로 표시해야 합니다.

```swift
class AnotherSubClass: SomeBaseClass {
  let string: String
  required init(string: String) {
    self.string = string
  }
  override class func printClassName() {
    print("AnotherSubClass")
  }
}
let metatype: AnotherSubClass.Type = AnotherSubClass.self
let anotherInstance = metatype.init(string: "some string")
```

### Any Type ('Any' 타입)

`Any` 타입은 다른 모든 타입의 값을 담을 수 있습니다. `Any` 는 다음 타입 중 어떤 것의 인스턴스에 대해서라도 '고정된 타입 (concrete type)' 으로 사용할 수 있습니다.

* 클래스, 구조체, 또는 열거체
* '메타 타입 (metatype)', 가령 `Int.sef` 같은 것
* 튜플, 성분의 타입은 어떤 것이든 상관 없음
* 클로저 또는 함수 타입

```swift
let mixed: [Any] = ["one", 2, true, (a, 5.3), { () -> Int in return 6 }]
```

`Any` 를 인스턴스에 대한 '고정된 타입 (concrete type)' 으로 사용할 때는, 먼저 그 인스턴스를 '알려진 (known)' 타입으로 변환해야 속성 또는 메소드에 접근할 수 있습니다. '고정된 타입' 이 `Any` 인 인스턴스는 원래의 동적 타입을 유지하고 있어서-`as`, `as?`, 또는 `as!` 같은, '타입-변환 연산자' 를 사용하여 해당 타입으로 변환할 수 있습니다. 예를 들어, 다음 처럼 '혼성 배열 (heterogeneous array)' 의 첫 번째 객체를 `String` 으로 '조건부 내림 변환 (conditionally downcast)' 하려면 `as?` 를 사용합니다:

```swift
if let first = mixed.first as? String {
  print("The first item, '\(first)', is a string.")
}
// "The first item, 'one', is a string." 을 출력합니다.
```

'변환 (casting)' 에 대한 더 많은 정보는, [Type Casting (타입 변환)]({% post_url 2020-04-01-Type-Casting %}) 을 참고하기 바랍니다.

`AnyObject` 프로토콜은 `Any` 타입과 비슷합니다. 모든 클래스는 암시적으로 `AnyObject` 를 준수합니다. 언어에서 정의하는, `Any` 와는 달리, `AnyObject` 는 '스위프트 표준 라이브러리' 에서 정의합니다. 더 많은 정보는, [Class-Only Protocols (클래스-전용 프로토콜)]({% post_url 2016-03-03-Protocols %}#class-only-protocols-클래스-전용-프로토콜) 과 [AnyObject](https://developer.apple.com/documentation/swift/anyobject) 를 참고하기 바랍니다.

> GRAMMAR OF AN ANY TYPE 부분 생략 - [Apple 홈페이지가 업데이트 되지 않아서 링크는 아직 없습니다.]

### Self Type ('Self' 타입)

`Self` 타입은 지정된 타입이라기 보다는, 타입의 이름을 알아야 하거나 반복하지 않아도 펀리하게 현재 타입을 참조하도록 해 주는 것입니다.

프로토콜 선언이나 프로토콜 멤버 선언에 있는, `Self` 타입은 이 프로토콜을 최종적으로 준수하는 타입을 참조합니다.

구조체나, 클래스, 또는 열거체 선언에 있는, `Self` 타입은 이 선언이 도입하는 타입을 참조합니다. 타입의 멤버를 위한 선언 내에 있는. `Self` 타입은 해당 타입을 참조합니다. 클래스 선언의 멤버 내에 있는, `Self` 는 다음의 경우에서만 나타날 수 있습니다:

* 메소드의 반환 타입으로써
* 읽기-전용 첨자 연산의 반환 타입으로써
* 읽기-전용 계산 속성의 타입으로써
* 메소드의 본문에서

예를 들어, 아래의 코드는 반환 타입이 `Self` 인 `f` 라는 인스턴스 메소드를 보여줍니다:

```swift
class Superclass {
  func f() -> Self { return self }
}
let x = Superclass()
print(type(of: x.f()))
// "Superclass" 를 출력합니다.

class Subclass: Superclass { }
let y = Subclass()
print(type(of: y.f()))
// "Subclass" 를 출력합니다.

let z: Superclass = Subclass()
print(type(of: z.f()))
// "Subclass" 를 출력합니다.
```

위 예제의 마지막 부분은 `Self` 가 참조하는 것이 `z` 값의 실행 시간 타입인 `Subclass` 이지, 변수 그 자체의 컴파일-시간 타입인 `Superclass` 가 아님을 보여줍니다.

'중첩 타입 (nested type)' 선언 내에 있는, `Self` 타입은 '가장 안쪽의 (innermost)' 타입 선언이 도입한 타입을 참조합니다.

`Self` 타입은 스위프트 표준 라이브러리에 있는 `type(of:)` 함수와 같은 타입을 참조합니다.[^Self-and-type-of] `Self.someStaticMember` 라고 작성해서 현재 타입의 멤버에 접근하는 것은 `type(of: self).someStaticMember` 라고 작성하는 것과 같습니다.

### Type Inheritance Clause (타입 상속 구절)

_타입 상속 구절 (type inheritance clause)_ 은 '이름 있는 타입' 이 상속하는 클래스와 '이름 있는 타입' 이 준수하는 프로토콜이 어떤 것인지를 지정하는데 사용합니다. '타입 상속 구절' 은 '콜론 (`:`)' 으로 시작하고, 그 뒤에 타입 식별자 목록이 옵니다.

클래스 타입은 상위 클래스를 단 하나 상속할 수 있으며 프로토콜은 몇 개라도 준수할 수 있습니다. 클래스를 정의할 때, 상위 클래스의 이름은 반드시 타입 식별자 목록의 맨 처음에 있어야 하며, 그 뒤에 클래스가 반드시 준수해야 하는 프로토콜은 몇 개라도 있을 수 있습니다. 클래스가 다른 클래스를 상속하지 않는 경우에는, 목록을 프로토콜로 시작할 수 있습니다. 클래스 상속에 대한 더 깊은 논의와 여러가지 예제를 보려면, [Inheritance (상속)]({% post_url 2020-03-31-Inheritance %}) 를 참고하기 바랍니다.

다른 이름 있는 타입[^other-named-type] 들은 프로토콜 목록 만을 상속받거나 준수할 수 있습니다. 프로토콜 타입은 다른 프로토콜을 몇 개라도 상속할 수 있습니다. 프로토콜 타입이 다른 프로토콜을 상속받을 때는, 다른 프로토콜에 있는 필수 조건들의 집합을 한데 모아서, 현재 프로토콜을 상속받는 어떤 타입이든 이 필수 조건들을 반드시 모두 준수해야 합니다.

열거체 정의에 있는 타입 상속 구절은 프로토콜 목록일 수도 있으나, 'case 값' 에 '원시 값' 을 할당하는 열거체인 경우, 해당 원시 값의 타입을 지정하는 단일한, 이름 있는 타입일 수도 있습니다. 타입 상속 구절을 사용하여 원시 값의 타입을 지정하는 열거체 정의에 대한 예제는, [Raw Values (원시 값)]({% post_url 2020-06-13-Enumerations %}#raw-values-원시 값) 을 참고하기 바랍니다.

> GRAMMAR OF A TYPE INHERITANCE CLAUSE 부분 생략 - [링크](https://docs.swift.org/swift-book/ReferenceManual/Types.html#ID456)

### Type Inference (타입 추론)

스위프트는 _타입 추론 (type inference)_ 을 광범위하게 사용하여, 코드에 있는 많은 변수와 표현식의 타입 또는 타입 일부를 생략하도록 해줍니다. 예를 들어, `var x: Int = 0` 라고 작성하는 대신에, 타입을 완전히 생략하여, `var x = 0` 이라고 작성할 수 있습니다-컴파일러는 `x` 가 `Int` 타입 값의 이름이라고 올바르게 추론합니다. 이와 비슷하게, 상황으로부터 온전한 타입을 추론할 수 있을 때는 타입 일부를 생략할 수 있습니다. 예를 들어, `let dict: Dictionary = ["A": 1]` 이라고 작성하면, 컴파일러는 `dict` 이 `Dictionary<String, Int>` 타입이라고 추론합니다.

위의 두 예제 모두에서, 타입 정보는 표현식 트리의 '말단 (leaves)' 부터 그 '근원 (root)' 까지 전달됩니다. 즉, `var x: Int = 0` 에 있는 `x` 의 타입은 먼저 `0` 의 타입을 검사한 다음 이 타입 정보를 근원 (변수 `x`) 까지 전달하여 추론합니다.

스위프트에서, 타입 정보는-근원에서 부터 말단까지-반대 방향으로도 흐릅니다. 다음 예제에서, 예를 들어 보자면, 상수 `eFloat` 에 대한 명시적인 타입 보조 설명 (`: Float`) 은 '수치 글자 값' 인 `2.71828` 이 `Double` 이 아니라 `Float` 이라는 추론 타입을 가지도록 합니다.

```swift
let e = 2.71828 // e 의 타입은 Double 이라고 추론됩니다.
let eFloat: Float = 2.71828 // eFloat 의 타입은 Float 입니다.
```

이것의 의미는 표현식에 있는 추론할 필요가 있는 일부 타입 또는 생략된 타입에 대한 모든 정보는 그 표현식의 타입-검사 또는 그것의 하위 표현식 중 하나의 타입-검사로부터 반드시 접근 가능해야 한다는 것입니다.

[^initializer-expression]: '초기자 표현식 (initializer expression)' 이란 타입에 있는 초기자를 `.init` 를 이용하여 명시적으로 호출하는 표현식을 말합니다. '초기자 표현식' 에 대한 정보는 [Initializer Expression (초기자 표현식)]({% post_url 2020-08-19-Expressions %}#initializer-expression-초기자-표현식) 을 참고하기 바랍니다.

[^other-named-type]: 스위프트에서 '이름 있는 타입 (named type)' 은 이 장 맨 처음에서 설명한 것처럼, 클래스, 구조체, 열거체, 그리고 프로토콜입니다. 즉, 여기서 '다른 이름 있는 타입 (other named types)' 들은 이 중에서 클래스를 제외한, 구조체, 열거체, 및 프로토콜을 말하는 것입니다.

[^Self-and-type-of]: 이 설명만 보면, `Self` 타입과 `type(of:)` 함수의 결과가 같으므로, `Self` 가 있을 필요는 없는 것처럼 보입니다. 하지만 `type(of:)` 함수는 특정 값의 동적 타입을 반환하는 함수로 컴파일 시간에 호출되지 않습니다.[^type-of] 반면, `f()` 함수의 정의에서 반환 값을 지정해줘야 하는데, 반환 값의 지정은 컴파일 시간에 해야 하므로 `type(of:)` 함수를 호출할 수 없습니다. 즉, `Self` 를 사용하면 컴파일 시간에 동적으로 변할 수 있는 타입의 현재 타입을 지정할 수 있습니다.

[^type-of]: 더 자세한 내용은 Apple 개발 문서의 [type(of:)](https://developer.apple.com/documentation/swift/2885064-type) 항목을 참고하기 바랍니다.
