## Patterns (패턴)

_패턴 (pattern)_ 은 '단일 값 또는 합성 값의 구조 (structure)' 를 표현합니다. 예를 들어, '`(1, 2)` 라는 튜플의 구조' 는 '두 원소를 쉼표로 구분한 목록' 입니다. '패턴' 은 어떤 하나의 특별한 값이라기 보다는 '값의 구조' 를 표현하기 때문에, 이는 다양한 값과 일치할 수 있습니다. 구체적인 사례를 보면, '`(x, y)` 라는 패턴' 은 '`(1, 2)` 라는 튜플' 및 '어떤 다른 두-원소의 튜플' 과도 일치합니다. 패턴과 값의 일치에 더하여, 합성 값의 일부나 전체를 추출하여 각 부분을 상수나 변수 이름에 연결할 수도 있습니다.

스위프트에는: '어떤 종류의 값과도 일치 성공하는 것', 그리고 '실행 시간에 특정 값과는 일치 실패할 수도 있는 것' 이렇게 두 종류의 '기초 패턴' 이 있습니다.

첫 번째 종류의 패턴은 '단순한 변수, 상수, 및 옵셔널 연결 (optional binding) 값의 해체 (destructuring)' 에 사용합니다. 이는 '와일드카드 (wildcard patterns) 패턴', '식별자 (identifier) 패턴', 및 '이들을 담은 어떤 값 연결 (value binding) 이나 튜플 (tuple patterns) 패턴' 을 포함합니다. 이 패턴들이 정해진 타입의 값만 일치하도록 구속하기 위해 이들에 '타입 보조 설명 (type annotation)' 을 지정할 수 있습니다.

두 번째 종류의 패턴은, 맞춰보려는 값이 실행 시간에 없을 수도 있는, '전체 패턴 맞춤 (full pattern matching)' 에 사용합니다. 이는 '열거체 case(enumeration case) 패턴', '옵셔널 (optional) 패턴', '표현식 (expression) 패턴', 및 '타입-변환 (type-casting) 패턴' 을 포함합니다. 이 패턴들은 '`switch` 문의 `case` 이름표' 나, '`do` 문의 `catch` 절', 또는 '`if`, `while`, `guard`, 및 `for`-`in` 문의 조건' 에 사용합니다.

### Wildcard Pattern (와일드카드 패턴)

_와일드카드 패턴 (wildcard pattern)_ 은 어떤 값과도 일치하고 무시하며 '밑줄 (`_`)' 로 구성합니다. 맞춰볼 값에 신경을 쓰고 싶지 않을 때 '와일드카드 패턴' 을 사용합니다. 예를 들어, 다음 코드는, 반복문의 각 회차에 대한 '현재의 범위 값' 을 무시하면서, `1...3` 이라는 '닫힌 범위' 를 반복합니다:

```swift
for _ in 1...3 {
  // 뭔가를 세 번 반복합니다.
}
```

> GRAMMAR OF A WILD PATTERN 부분 생략 - [링크](https://docs.swift.org/swift-book/ReferenceManual/Patterns.html#ID420)

### Identifier Pattern (식별자 패턴)

_식별자 패턴 (identifer pattern)_ 은 어떤 값과도 일치하며 '일치한 값을 변수나 상수 이름에 연결' 합니다. 예를 들어, 다음 상수 선언의, `someValue` 는 `Int` 타입의 값 `42` 와 일치하는 '식별자 패턴' 입니다:

```swift
let someValue = 42
```

일치 성공하면, `42` 라는 값을 `someValue` 라는 상수 이름에 연결 (할당) 합니다.

변수나 상수 선언 왼쪽의 패턴이 식별자 패턴일 때는, 암시적으로 '식별자 패턴' 이 '값-연결 패턴의 하위 패턴' 입니다.

> GRAMMAR OF A IDENTIFIER PATTERN 부분 생략 - [링크](https://docs.swift.org/swift-book/ReferenceManual/Patterns.html#ID421)

### Value-Binding Pattern (값-연결 패턴)

_값-연결 패턴 (value-binding pattern)_ 은 '일치 값' 을 '변수나 상수 이름에 연결' 합니다. 값-연결 패턴에서 일치 값을 상수 이름에 연결하는 건 `let` 키워드로 시작하고; 변수 이름에 연결하는 건 `var` 키워드로 시작합니다.

값-연결 패턴 안의 '식별자 패턴' 은 '새로 이름 붙인 변수나 상수를 자신과 일치하는 값에 연결' 합니다. 예를 들어, 튜플의 원소를 분해해서 '각 원소 값을 관련 식별자 패턴에 연결' 할 수 있습니다.

```swift
let point = (3, 2)
switch point {
  // x 와 y 를 point 원소에 연결합니다.
case let (x, y):
  print("The point is at (\(x), \(y)).")
}
// "The point is at (3, 2)." 를 인쇄합니다.
```

위 예제에서는, `let` 을 `(x, y)` 튜플 패턴의 각 식별자 패턴으로 배포합니다.[^distribute] 이 동작 때문에, `case let (x, y):` 와 `case (let x, let y):` 라는 `switch` 문 case 절' 들이 일치하는 값은 똑같습니다.

> GRAMMAR OF A VALUE-BINDING PATTERN 부분 생략 - [링크](https://docs.swift.org/swift-book/ReferenceManual/Patterns.html#ID422)

### Tuple Pattern (튜플 패턴)

_튜플 패턴 (tuple pattern)_ 은 '쉼표로 구분한 0개 이상의 패턴을, 괄호로 테두리 친, 목록' 입니다. '튜플 패턴' 은 '관련 튜플 타입 값' 과 일치합니다.

튜플 타입이 '정해진 종류의 튜플 타입과 일치하도록 구속' 하기 위해 '타입 보조 설명' 을 사용할 수 있습니다. 예를 들어, '`let (x, y): (Int, Int) = (1, 2)` 라는 상수 선언' 에 있는 '`(x, y): (Int, Int)` 라는 튜플 패턴' 은 '원소 둘 다 `Int` 타입인 튜플 타입' 하고만 일치합니다.

튜플 패턴을 '`for`-`in` 문의 패턴' 또는 '변수나 상수 선언의 패턴' 으로 사용할 때는, 와일드카드 패턴, 식별자 패턴, 옵셔널 패턴, 또는 '이들을 담은 다른 튜플 패턴' 만을 담을 수 있습니다. 예를 들어, 다음 코드는 `(x, 0)` 라는 튜플 패턴의 원소 `0` 이 '표현식 (expression) 패턴' 이기 때문에 유효하지 않습니다:

```Swift
let points = [(0, 0), (1, 0), (1, 1), (2, 0), (2, 1)]
// 이 코드는 유효하지 않습니다.
for (x, 0) in points {
    /* ... */
}
```

'단일 원소를 담은 튜플 패턴 주위의 괄호' 는 아무 효과도 없습니다. 패턴은 단일 원소인 타입의 값과 일치합니다. 예를 들어, 다음은 서로 '동치 (equivalent)' 입니다:

```swift
let a = 2        // a: Int = 2
let (a) = 2      // a: Int = 2
let (a): Int = 2 // a: Int = 2
```

### Enumeration Case Pattern (열거체 case 패턴)

_열거체 case 패턴 (enumeration case pattern)_ 은 '기존 열거체 타입의 case 값과 일치' 합니다. 열거체 case 패턴은 '`switch` 문의 case 이름표' 와 '`if`, `while`, `guard`, 및 `for`-`in` 문의 조건' 에 나타납니다.

'맞춰보려는 열거체 case 값' 이 '어떤 결합 값' 을 가진 경우, 반드시 '각 결합 값마다 한 원소를 담은 튜플 패턴' 을 '관련 열거체 case 패턴' 에 지정해야 합니다. '결합 값을 담은 열거체 case 값' 과 맞춰보기 위해 '`switch` 문' 을 사용하는 예제는, [Associated Values (결합 값)]({% post_url 2020-06-13-Enumerations %}#associated-values-결합-값) 부분을 참고하기 바랍니다.

열거체 case 패턴은 '해당 case 값을 옵셔널로 포장한 값' 과도 일치합니다. 이런 단순화 구문은 '옵셔널 패턴' 을 생략할 수 있게 합니다. `Optional` 이 열거체로 구현되어 있기 때문에, 동일한 swift 문에 '열거체 타입의 case 값' 으로 `.none` 과 `.some` 을 나타낼 수 있음을 기억하기 바랍니다. 

```swift
enum SomeEnum { case left, right }
let x: SomeEnum? = .left
switch x {
case .left:
  print("Turn left")
case .right:
  print("Turn right")
case nil:
  print("Keep going straight")
}
// "Turn left" 를 인쇄합니다.
```

### Optional Pattern (옵셔널 패턴)

_옵셔널 패턴 (optional pattern)_ 은 '`Optional<Wrapped>` 열거체의 `some(Wrapped)` case 값으로 포장한 값' 과 일치합니다. 옵셔널 패턴은 '식별자 패턴과 그 뒤의 물음표' 로 구성하며 '열거체 case 패턴과 동일한 장소' 에 나타냅니다.

옵셔널 패턴은 '`Optional` 열거체 case 패턴의 수월한 구문 표현 (syntatic sugar)' 이기 때문에, 다음은 서로 '동치 (equivalent)' 입니다:

```swift
let someOptional: Int? = 42
// 열거체 case 패턴으로 맞춰보기
if case .some(let x) = someOptional {
  print(x)
}

// 옵셔널 패턴으로 맞춰보기
if case let x? = someOptional {
  print(x)
}
```

옵셔널 패턴은, `for`-`in` 문에 있는, `nil`-아닌 원소에만 반복문 본문을 실행하면서, 옵셔널 값 배열을 반복하는 '편의' 를 제공합니다:

```swift
let arrayOfOptionalInts: [Int?] = [nil, 2, 3, nil, 5]
// nil-아닌 값 하고만 일치합니다.
for case let number? in arrayOfOptionalInts {
  print("Found a \(number)")
}
// Found a 2
// Found a 3
// Found a 5
```

### Type-Casting Patterns (타입-변환 패턴)

'타입 변환 (type-casting) 패턴' 에는, '`is` 패턴과 `as` 패턴' 이라는, 두 가지가 있습니다. `is` 패턴은 '`switch` 문의 case 이름표' 에서만 나타납니다. `is` 패턴과 `as` 패턴의 형식은 다음과 같습니다:

&nbsp;&nbsp;&nbsp;&nbsp;is `type-타입`<br />
&nbsp;&nbsp;&nbsp;&nbsp;`pattern-패턴` as `type-타입`

'`is` 패턴' 은 '실행 시간의 해당 값 타입이 `is` 패턴의 오른쪽에서 지정한 타입-또는 그 타입의 하위 클래스-와 똑같은 경우' 에 일치합니다. `is` 패턴은 둘 다 타입을 변환한다는 점에서는 `is` 연산자와 동작이 같지만 반환 타입을 버립니다.

'`as` 패턴' 은 '실행 시간의 해당 값 타입이 `as` 패턴의 오른쪽에서 지정한 타입-또는 그 타입의 하위 클래스-와 똑같은 경우' 에 일치합니다. 일치 성공하면, 일치한 값의 타입을 `as` 패턴의 오른쪽에서 지정한 _패턴 (pattern)_ 으로 '변환 (cast)' 합니다.

`is` 와 `as` 패턴으로 값을 맞춰보는 `switch` 문의 사용 예제는, [Type Casting for Any and AnyObject ('Any' 와 'AnyObject' 를 위한 타입 변환)]({% post_url 2020-04-01-Type-Casting %}#type-casting-for-any-and-anyobject-any-와-anyobject-를-위한-타입-변환) 부분을 참고하기 바랍니다.

> GRAMMAR OF A TYPE CASTING PATTERN 부분 생략 - [링크](https://docs.swift.org/swift-book/ReferenceManual/Patterns.html#ID425)

### Expression Pattern (표현식 패턴)

_표현식 패턴 (expression pattern)_ 은 '표현식의 값' 을 나타냅니다. 표현식 패턴은 '`switch` 문의 case 이름표' 에서만 나타납니다.

'표현식 패턴이 나타낸 표현식' 은 '스위프트 표준 라이브러리의 `~=` 연산자'[^compare-range] 를 써서 '입력 (input) 표현식 값' 과 비교합니다. `~=` 연산자가 `true` 를 반환하면 일치 성공입니다. 기본적으로, `~=` 연산자는 동일한 타입의 두 값을 `==` 연산자로 비교합니다. 이는, 아래 예제에서 보는 것처럼, 범위 안에 값이 담겨 있는 지를 검사함으로써, 값을 값의 범위와 맞춰볼 수 있습니다.

```swift
let point = (1, 2)
switch point {
case (0, 0):
  print("(0, 0) is at the origin.")
case (-2...2, -2...2):
  print("(\(point.0), \(point.1)) is near the origin.")
default:
  print("The point is at (\(point.0), \(point.1)).")
}
// "(1, 2) is near the origin." 를 인쇄합니다.
```

`~=` 연산자는 '사용자 정의 표현식 맞춤(expression mating) 동작을 제공' 하도록 '중복정의 (overload)' 할 수 있습니다. 예를 들어, 위 예제를 재작성하면 '`point` 표현식' 과 '점의 문자열 표현' 을 비교할 수 있습니다.

```swift
// 문자열과 정수를 맞춰보기 위해 ~= 연산자 중복정의하기
func ~= (pattern: String, value: Int) -> Bool {
  return pattern == "\(value)"
}
switch point {
case ("0", "0"):
  print("(0, 0) is at the origin.")
default:
  print("The point is at (\(point.0), \(point.1)).")
}
// "The point is at (1, 2)." 를 인쇄합니다.
```
